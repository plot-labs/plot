# 0001. Monorepo With Generated Apps

## Status

Accepted

## Decision

Plot uses a monorepo:

- `apps/web` is generated by `create-next-app`.
- `apps/api` is generated by Spring Initializr.
- Shared TypeScript packages live under `packages`.
- Product, architecture, API, and operations notes live under `docs`.

The backend and agent runtime stay in the same Spring Boot process for v1.

## Rationale

This keeps the product small enough to build while still making the backend architecture visible for portfolio review. The generated app foundations avoid hand-rolled framework setup, while the domain modules make Plot's core loop explicit:

```txt
Writing Blocks -> Signals -> Brief -> Angle -> Content Pack -> Learn -> Follow-up
```

## Consequences

- Frontend and backend can evolve independently but ship from one repo.
- The API is the source of truth for domain state and agent trace.
- The frontend should consume a generated API client once OpenAPI is wired.
- Landing pages start inside `apps/web`; `apps/landing` can be split later if SEO or marketing operations justify it.
